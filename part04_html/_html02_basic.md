## HTML简介

- HTML是HyperText Mark-Up Language 的首字母简写，意思是**超文本标记语言**
  - 超：超链接
  - 文本: 用纯文本编辑器所写的东西就是文本。纯文本编辑器只能写字，它只能存储数据，与数据无关的格式，颜色，字体，图片等都不能存储
    - word文档是复文本，因为word文档里面不仅可以存文字，还可以存图片等
  - 标记：标识网页中的不同内容，告诉浏览器这些内容是什么
    - <标签名></标签名>
- 如果你的文件希望被浏览器打开，需要将扩展名保存为html
  
- <html></html>是根标签，在网页中有且只有一个，网页中的所有内容都要写根标签的里边。（标签和元素是同一个东西）

  - head （网页的头部）和body是html下的两个子标签
  - head中写的东西在网页中是看不见的，是给浏览器/搜索引擎看的，为了让浏览器快速解析网页
    - head下的标签
      1. meta标签用来设置网页的元数据，这里meta用来设置网页的字符集，避免乱码问题
      2. title标签，title标签下的内容会显示在浏览器的标题栏，搜索引擎主要会根据title中的内容来判断网页的主要内容
  - body是网页的主体，网页中所有的可见内容都应该写在body中

- 注释： 不会在网页中显示，可以在源码中查看。注释还可以将不希望显示的内容隐藏

  - 注释不能嵌套的 （不能在多行注释中再写多行注释

  ```
  <!-- 中间写注释 -->
  ```

- 自结束标签： 单个出现的标签

  - <input />, <img />, <br />

- 标签中的属性

  - 在标签中可以设置属性，属性是名值对结构 （x=y）
  - 属性用来设置标签中的内容如何显示
  - 属性和标签名或其他属性应该使用空格隔开
  - 属性应该根据文档中的规定来编写，有些属性有属性值，有些属性没有属性值，没有属性值的属性直接写属性名即可，如果有属性值应该用引号引起来

- 文档声明(doctype): 声明当前网页的版本

  - 网页会有不同的版本，e.g. HTML4, XHTML2.0, HTML5
  - 文档声明用来告诉浏览器当前网页的版本，写在网页的第一行
  - html5的文档声明：<!DOCTYPE html> ，这个内容就是用来告诉浏览器网页的规范，不属于网页中的内容，写在html根标签的外面（网页的第一行）

## 实体 （书写html中的特殊字符)

- 实体的语法：&开头，；结尾，中间是实体的名字
- 实体的种类：
  1. 空格
     - 在代码中空多个空格，在渲染成网页时只会显示一个空格，如果想显示多个空格，可以使用空格的字符实体:**\&nbsp;** （因为在我们写程序的时候经常用空格、换行、缩进来对我们的代码进行格式化的操作）
  2. 大于号和小于号
     - 在html中大于号和小于号是用来标识标签的，所以不能直接书写字母两侧的大于号和小于号
     - 大于号：**\&gt;**
     - 小于号： **\&lt;**
  3. 版权声明符号：
     - &copy；

## 标签

1. meta标签 （用来设置网页的元数据），自结束标签

   - 元数据是给浏览器和搜索引擎看到

   - 常用属性：

     - name：指定的数据的名称

       - 常见name属性：

         1. keywords：表示网站的关键字，可以同时指定多个关键字，关键字间使用逗号隔开

         2. description：用于指定网站的描述 （网站的描述会显示在搜索引擎的搜索的结果中）

         3. http-equiv：将页面重定向到另一个网站

            ```
             <meta http-equiv="refresh" content="3;url=https://www.mozilla.org"> 
             // 3表示3秒之后跳转，后面的网址表示要跳转到的页面
            ```

     - content：指定的数据的内容 （数据的值）

     - charset：指定网页的字符集

2. title标签的内容会作为搜索结果的超链接上的文字显示   

3. 语义化标签：

   - 在网页中HTML专门用来负责网页的结构，所以在使用html标签时，应该**关注的是标签的语义**（标签代表的是什么意思），而不是它的样式 （它的样式可以根据css的样式来进行更改）

   - 标签种类

     3.1 标题标签：h1 ~ h6 一共有六级标题

     ```
     <h1>一级标题</h1>
     <h2>二级标题</h2>
     <h3>三级标题</h3>
     <h4>四级标题</h4>
     <h5>五级标题</h5>
     <h6>六级标题</h6>
     ```

     - 从h1~h6**重要性递减**，h1最重要，h6最不重要 (关注的是标题的重要性，而不是标题的大小)
     - h1在网页中的**重要性仅次于title标签**（搜索引擎首先看的是title，接下来就是看h1标签），一般情况下一个页面中**只会有一个h1**，一般情况下标题标签只会使用到h1~h3，h4~h6很少用
     - 标题标签都是**块元素** （页面中独占一行的元素称为块元素 block element)

     3.2 hgroup标签用来**为标题分组**，可以将一组**相关的**标题同时放入到hgroup (是因为两个标题有联系才会放一起)

     ```
     <hgroup>
     	<h1>回乡偶书二首</h1>
     	<h2>其一</h2>
     </hgroup>
     ```

     3.3 p标签：表示页面中的一个段落 

     - p也是一个**块元素**

     ```
     <p>在p标签中的内容就表示一个段落</p>
     ```

     3.4 em标签：表示语音语调的一个加重 (emphasis)，主要表示的是语气的加重

     - 在页面中不会独占一行的元素称为**行内元素**/内联元素（inline element）, em标签是行内元素

     ```html
     <p>今天天气<em>真</em>不错！</p>
     ```

     3.5 strong标签：

     - strong表示强调，**重要内容**！

     ```html
     <p>你今天必须要<strong>完成作业</strong>！</p>
     ```

     3.6 blockquote 块引用标签

     ```html
       鲁迅说：
       <blockquote>
       这句话我是从来没有说过的！
       </blockquote>
     ```

     - 表示一个长引用，一般引用的内容比较多的时候使用
     - 是块元素

     3.7 q标签

     ```
     子曰<q>学而时习之，乐呵乐呵！</q>
     ```

     - 是一个短引用, 会自动加引号
     - 是个行内元素

4. 换行标签 （break return） \<br/> 

5. 结构化的语义标签 （主要用来对网页进行布局的，将网页分成一块一块）（前7个标签使用的都不多)

   5.1 header 表示网页的头部，网页中可以有多个header

   5.2  main 表示网页的主体部分(一个页面中**只会有一个**main)

   5.3 footer 表示网页的底部

   5.4 nav 表示网页中的导航

   5.5 aside 和主体相关的其他内容（侧边栏）

   5.6 article 表示一个独立的文章

   5.7 section 表示一个独立的区块，上边的标签都不能表示时使用section

   5.8 div 没有语义，就用来表示一个区块，目前来讲div还是我们主要的布局元素 (**块元素**)

   5.9 span **行内元素**，没有任何的语义，一般用于在网页中选中文字

6. 列表

   - 在html中也可以创建列表，html列表一共有三种：

     6.1 有序列表： 使用ol标签来创建无序列表， 使用li表示列表项 （每个表项前面会显示数字）

     ```html
     <ol>
     	<li>结构</li>
     	<li>表现</li>
     	<li>行为</li>
     </ol>
     ```

     6.2 无序列表： 使用ul标签来创建无序列表， 使用li表示列表项 （每个表项前面会显示一个点），ul实际应用最多，网页中的导航菜单都可以用ul来做

     ```html
     <ul>
     	<li>结构</li>
     	<li>表现</li>
     	<li>行为</li>
     </ul>
     ```

     6.3  定义列表 （为一些东西下定义的）：使用dl标签来创建一个定义列表，使用dt来表示定义的内容，使用dd来对内容进行解释说明 (一个dt可以对应多个dd)

     ```html
     <dl>
     	<dt>结构</dt>
     	<dd>结构表示网页的结构，结构用来规定网页中哪里是标题，哪里是段落</dd>
     	<dd>结构表示网页的结构，结构用来规定网页中哪里是标题，哪里是段落</dd>
     	<dd>结构表示网页的结构，结构用来规定网页中哪里是标题，哪里是段落</dd>
     </dl>
     ```

   - 为确保同样的东西在不同的浏览器中有相同的效果，所以在用的时候会去掉ol中的数字和ul中前面的点，因此在使用中两种列表的使用方式区别不大

   - 列表之间可以互相嵌套

7. 超链接标签

   - 超链接可以让我们从一个页面跳转到其他页面，或者是当前页面的其他的位置

   - 使用 a 标签来定义超链接

   - a标签的属性

     1. **href** 指定跳转的目标路径，值可以是一个外部网站的地址，也可以写一个内部页面的地址 （会通过颜色区别哪些地址访问过，哪些地址没访问过）

        - 直接将href属性设置为#，这样点击这个超链接页面不会发生跳转，而是转到当前页面顶部的位置 (无论在哪里使用#，#总是用来回到顶部)

          ```html
          <a href="#">这是一个新的超链接</a>
          ```

        - 可以跳转到页面的指定位置 (锚点) ，只需将href属性设置 #目标元素的id属性值 

          ```html
          <a href="#bottom">去底部</a>
          <a id="bottom" href="#">回到顶部</a>
          ```

        - 可以使用 javascript:; 来作为href的属性，此时点击这个超链接什么也不会发生 (这是在开发中用来占位的，需要有一个超链接，但是页面暂时还没确定，使用这种方式来占位表示之后会链接一个页面)

          ```html
          <a href="javascript:;">这是一个新的超链接</a>
          ```

     2. **target** 用来指定超链接打开的位置

        - 可选值：
              _self 默认值 在当前页面中打开超链接 （不写target属性就是这个值）
              _blank 在一个新的标签中打开超链接

     3. **id**：唯一不重复的

        - 每一个标签都可以添加一个id属性
        - id属性就是元素的唯一标识，同一个页面中不能出现重复的id属性
        - id都是字母开头的，且id是区分大小写的

   - 超链接是也是一个**行内元素**，在a标签中**可以嵌套除它自身外的任何元素** （通常块元素中什么都可以放，但是行内元素中是不能放块元素的）

   - 相对路径

     - 当我们需要跳转一个服务器内部的页面时，**一般我们都会使用相对路径**
       - 相对路径都会使用./或../开头，./可以省略不写，如果不写./也不写../则就相当于写了./
         - ./ 表示**当前文件**所在的目录 （你写这个路径的那个文件）
         - ../ 表示当前文件所在目录的上一级目录

8. 图片标签

   - 图片标签用于向当前页面中引入一个外部图片

   - img标签是一个自结束标签

   - img属于**替换元素**

     - 介于块元素和行内元素之间，具有两种元素的特点 （但是img不独占一行）
     - 替换元素显示的内容并不是直接写的内容本身，而是引入的外部资源

   - 标签属性：

     1. src:  属性指定的是图片路径（路径规则和超链接是一样的，可以引入内部图片也可以引入外部图片，但一般不建议用外部图片，网站可能不允许用某些图片，而且如果网站将图片给删除了就显示不出来）

     2. alt：图片的描述，这个描述**默认情况下不会显示**，有些浏览器会在图片无法加载时显示，搜索引擎会根据alt中的内容来识别图片，如果不写alt属性则图片不会被搜索引擎所收录

     3. width 图片的宽度 (单位是像素)， height 图片的高度

        - 宽度和高度中如果只修改了一个，则另一个会**等比例缩放**
        - 一般情况在pc端，不建议修改图片的大小，需要多大的图片就裁多大（用多大的图就切多大的图，大图缩小浪费内存，小图放大会失真），但是在移动端，经常需要对图片进行缩放（大图缩小，移动端通常像素会高一些，如果放小图片会不清晰，通常会将一个大图片放小，很少会将小图放大）

        ```html
        <img src="./img/1.gif" alt="松鼠">
        <img width="200"  src="https://d2ggl082rr1mkp.cloudfront.net/category/IronMan_preview_1521810286_220_310.jpeg" alt="钢铁侠">
        ```

   - 图片格式
     1. jpeg(jpg)
        - 支持的颜色比较丰富，不支持透明效果，不支持动图
        - 一般用来显示照片
     2. gif
        - 支持的颜色比较少，支持简单透明，支持动图
        -  颜色单一的图片，动图
     3. **png**
        - 支持的颜色丰富，支持复杂透明，不支持动图
        - 颜色丰富，复杂透明图片（专为网页而生）
     4. webp
        - 这种格式是谷歌新推出的专门用来表示网页中的图片的一种格式
        - 它具备其他图片格式的所有优点，而且文件还特别的小 （谷歌的压缩算法比较先进），还支持动图
        - 缺点：兼容性不好 （ie等老的浏览器都不支持)
     5. base64 （并不是一种图片格式，希望图片加载速度快的时候用，使用场景不多）
        - 将图片使用base64编码，这样可以将图片转换为字符，通过字符的形式来引入图片
        - base64是对数据进行了加密
        - 一般都是一些需要和网页一起加载的图片才会使用base64 （网页在加载的时候分几部分加载，网页本身是有代码的，浏览器向服务器发送第一次请求请求网页本身的代码，但是网页中还会引入一些外部资源，这些外部资源浏览器还需要向服务器单独发送请求加载这些外部资源，有多少外部资源还需要再发送多少次请求加载资源）
   - 选择图片的原则
     - 效果一样用小的 （小的会有比较快的加载速度）
     - 效果不一样用效果好的

9. 内联框架 iframe （使用不多）

   - 用于向当前页面中引入一个其他页面 (一个网页中嵌套另一个网页)

   - iframe是一个成对出现的标签，有开始也有结束

   - 搜索引擎通常不会爬内联框架中的内容

   - iframe的属性

     1. src 指定要引入的网页的路径
     2. frameborder 指定内联框架的边框 （如果不写会默认在网页和内联框架之间有一个边框，如果向去除这个边框要将frameborder设置为0）

     ```html
     <iframe src="https://www.qq.com" width="800" height="600" frameborder="0"></iframe>
     ```

10. 音频标签

    - audio 标签用来向页面中引入一个外部的音频文件的，音视频文件引入时，默认情况下不允许用户自己控制播放停止

    - audio标签的属性

      1. controls 是否允许用户控制播放，这个属性不需要给值，如果想让用户控制就将这个属性写上就可以了 

      2. autoplay 音频文件是否自动播放，如果设置了autoplay 则音乐在打开页面时会自动播放，但是目前来讲大部分浏览器都不会自动对音乐进行播放

      3. loop 音乐是否循环播

         ```html
         <audio src="./source/audio.mp3" controls autoplay loop></audio
         ```

    - 除了通过src来指定外部文件的路径以外，还可以通过source标签来指定文件的路径 （通常使用这种方式来引入音频）

      - ie8并不支持audio标签

      - 我们希望ie8如果不支持这个标签可以显示出提示来，使用source标签来指定文件路径ie8也是不支持的，但是这种写法可以通过写注释的方式，让这个注释显示在网页上，给出错误信息提示 （不支持audio时候，会自动将audio和source的标签都忽略了，但是标签能忽略文字忽略不了会显示出来）

      - 支持source标签的会显示播放器，不支持source标签的会显示提示文字

      - source标签可以同时写多个，因为不同的浏览器支持的音频文件的格式可能是不一样的，会优先使用第一个source，如果第一个source播放不了会依次使用下面的，有效地解决了兼容的问题（现在这种情况比较少，之前比较多）

      - embed标签可以让ie8中的音频也能正常播放，但是这个标签不好用，必须要指定width和height才能让这个标签正常显示，并且需要指定文件的类型 （将embed标签放在最后是为了让其他浏览器不使用这个标签显示，但是如果遇到ie8音频还能播放出来）

        ```HTML
        <audio controls>
        	<!-- 对不起，您的浏览器不支持播放音频！请升级浏览器！ -->
        	<source src="./source/audio.mp3">
        	<source src="./source/audio.ogg">
        	<embed src="./source/audio.mp3" type="audio/mp3" width="300" height="100">
        </audio>
        ```

11. 视频标签

    - 使用video标签来向网页中引入一个视频，使用方式和audio基本上是一样的

    - 因为音频和视频是比较占内存的，所以通常我们不会将音频和视频放到服务器上，而是会专门买第三方的产品放在音频和视频再将资源引用到我们的服务器上（买内存和网络的成本相对较低）

      ```html
      <video controls>
      	<source src="./source/flower.webm">
      	<source src="./source/flower.mp4">
      	<embed src="./source/flower.mp4" type="video/mp4">
      </video>
      ```

    - 可以将我们的视频放到一些视频网站上再引用，这样就不需要买放视频的内存，而且这样做相当于在访问第三方网站的服务器 （使用iframe），对我们本地的服务器来说是没有压力的

      ```html
      <iframe frameborder="0" src="https://v.qq.com/txp/iframe/player.html?vid=b00318l66nt" allowFullScreen="true" width="500" height="300"></iframe>
      ```

      

## 块元素和行内元素

- 块元素（block element）

  - 在网页中一般通过块元素来对页面进行布局 (对网页进行宏观的布局)
  - p元素中不能放任何的块元素
  - 块元素中基本上什么都能放 （行内元素和块元素都可以放）

- 行内元素（inline element）

  - 行内元素主要用来包裹文字

- 注：

  1. 一般情况下会在块元素中放行内元素，而不会在行内元素中放块元素

  2. 浏览器在解析网页时，会自动对网页中不符合规范的内容进行修正

     - 我们可以通过查看 f12->elements 看到我们的代码加载到内存中浏览器修正之后的样子
     - 修正的过程会影响浏览器的性能，并且浏览器的修正不能保证符合我们的预期

     - 浏览器对不规范内容的修正，比如：
       - 标签写在了根元素的外部
       - p元素中嵌套了块元素
       - 根元素中出现了除head和body以外的子元素

## 其它

- 进制 （进制本质上就是数数的方式）

  - 进制种类
    - 十进制：满10进1 （单位数字只有10个）
    - 二进制：满2进1 （单位数字只有两个）,计算机底层的进制
      - 所有数据在计算机底层都会以二进制的形式保存
      - 可以将内存想象为要给有多个小格子的容器，每个小格子中可以存储一个1或一个0，这一个小格子在内存中被称为1位（bit）
      - 8 bit = 1byte (字节)， 1024byte = 1kb （千字节），1024kb = 1 mb（兆字节），1024 mb = 1 gb（吉字节），1024gb = 1tb（特字节）， 1024tb = 1pb
      - 在计算机中最小操作单位是byte， 不能单独操作一个bit
    - 八进制：满8进1 （单位数字只有8个），很少用
    - 十六进制：满16进1 （单位数字有16个：0-9，a-f）：一般要显示一个二进制数字时，都会转换为十六进制）

- 字符编码

  - 我们的所有数据在计算机中存储时都是以二进制形式存储的，文字也不例外。所以一段文字在存储到内存中，都需要转换为二进制编码。当我们读取这段文字时，计算机会将编码转换为字符，供我们阅读

  - 编码：将字符转换为二进制码的过程称为编码

  - 解码：将二进制码转换为字符的过程称为解码

  - 字符集：编码和解码所采用的规则称为字符集

  - 乱码问题：如果编码和解码所采用的字符集不同就会出现乱码问题

  - 常见的字符集：

    - ASCII：（一个7位，共有128个字符）
    - ISO88591 (欧洲人用的的，8位，256个字符)
    - GB2312 （国标，中国的字符集）
    - GBK （对国标的扩展）
    - UTF-8  （万国码，包含世界上所有的语言和符号），开发时使用的字符集都是utf-8

  - 在网页中我们通过meta标签设置网页的字符集，避免乱码问题

    ```
    <meta charset="UTF-8">
    ```
